# Что это за гайд

Вчера я с нуля собирал отдельную виртуальную Linux-машину под AI-агентов и решил оформить весь процесс в нормальный пошаговый гайд.

Идея такая: не ставить всё напрямую на основную Windows-систему, а поднять отдельный Ubuntu Server в Hyper-V, чтобы получить изолированную и стабильную среду для AI-инструментов.

В гайде разобран полный путь:

* установка Ubuntu Server 24.04 LTS в Hyper-V;
* настройка SSH и VS Code Remote SSH;
* установка Docker;
* настройка статического IP;
* развёртывание системного VPN через sing-box в TUN mode;
* проверка, что весь трафик Ubuntu и Docker идёт через VPN;
* установка Hermes Agent;
* базовая подготовка машины под AI-агентов, GPT/Gemini/OpenRouter/Claude и другие AI-инструменты.

В итоге получается отдельный AI-сервер внутри виртуальной машины:

```text
Windows Host
  ↓
Hyper-V
  ↓
Ubuntu Server
  ↓
Docker
  ↓
System-wide VPN
  ↓
Hermes Agent / AI tools
```

Такой подход удобен тем, что вся AI-инфраструктура живёт отдельно от основной системы. Если что-то сломается, можно откатиться через checkpoint или экспортировать готовую VM как шаблон.

Гайд больше рассчитан на новичков: я специально оставил команды, пояснения и типичные проверки после каждого важного этапа. Личные данные, IP, VPN-ключи и API-ключи в инструкции заменены на placeholders, чтобы её можно было безопасно адаптировать под свою конфигурацию.
